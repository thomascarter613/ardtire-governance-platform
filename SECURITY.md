# Security Policy

## Supported Versions

Only the current deployed version of the Ardtire Governance Platform receives active security support.

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub Issues.**

The Ardtire Governance Platform handles sensitive institutional governance records.
All security disclosures are handled privately.

Email the maintainer directly:

**Thomas J. Carter**
thomas.carter@appliedinnovationcorp.com

Include in your report:
- Description of the vulnerability and its potential impact
- Step-by-step reproduction instructions
- Relevant logs, screenshots, or proof-of-concept material
- Your suggested severity assessment

## Response Commitment

- Acknowledgement within **48 hours**
- Status update within **7 days**
- Coordinated disclosure before any public announcement

## Disclosure Policy

Private disclosure only. Public disclosure occurs only after a fix has been deployed
and affected parties have been notified.

## Scope

**In scope:**
- Authentication and session management vulnerabilities
- Authorization bypass or privilege escalation
- Injection vulnerabilities (SQL, command, etc.)
- Sensitive data exposure
- Governance record integrity violations
- Audit log integrity violations

**Out of scope:**
- Denial of service without significant institutional impact
- Social engineering attacks
- Physical security
